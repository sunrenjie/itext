Part I: Document Generation

Chapter 5: Tables

Introduction
A Table is a Rectangle that contains Cells, ordered in some kind of matrix.
The matrix of a table is not necessarily an m x n-matrix. It can contain holes or cells that are bigger than the unit. Believe me or not, but it took some serious thinking to make this as userfriendly as possible. I hope you wil find the result quite simple (I love simple solutions, especially for complex problems). I didn't want it to be something as complex as the Java GridBagLayout.

This is how you create a simple table (see Chap0501.java):
      Table table = new Table(3);
      table.setBorderWidth(1);
      table.setBorderColor(new Color(0, 0, 255));
      table.setCellpadding(5);
5.  table.setCellspacing(5);
      Cell cell = new Cell("header");
      cell.setHeader(true);
      cell.setColspan(3);
      table.addCell(cell);
10.  table.endHeaders();
      cell = new Cell("example cell with colspan 1 and rowspan 2");
      cell.setRowspan(2);
      cell.setBorderColor(new Color(255, 0, 0));
      table.addCell(cell);
15.  table.addCell("1.1");
      table.addCell("2.1");
      table.addCell("1.2");
      table.addCell("2.2");
      table.addCell("cell test1");
20.  cell = new Cell("big cell");
      cell.setRowspan(2);
      cell.setColspan(2);
      table.addCell(cell);
      table.addCell("cell test2");
25.  document.add(table);

The result of this code is a table like this (see Chap0501.pdf):
header
example cell with colspan 1 and rowspan 2 1.1 2.1
1.2 2.2
cell test1 big cell
cell test2
