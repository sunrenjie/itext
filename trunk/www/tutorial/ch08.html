Part II: Other document formats

Chapter 8: RTF
The RTF package
The RTF package is an extension to the base iText package and allows iText to output Rich Text files in addition to PDF files. This package was written by Mark Hall.

Most of the PDF features are available but there are some features which are not supported in the RTF package.

Two different writers exist:
  • The RtfWriter is the older one and is completely stable and well tested. Only bugfixes will be applied to this writer, no new functionality. It will be deprecated end of 2005 and removed end of 2007.
  • The RtfWriter2 is the new writer. It is reasonably stable and tested. The writer is where the active development will take place. If you are starting from scratch, use this writer.
Generating a RTF document
You generate RTF documents the same way you generate PDF documents. The 5 basic steps are the same. The only change is that you use a RtfWriter2 instead of the usual PdfWriter in Step 2.
Download the source code for the Hello World example: Chap0801.java

Step 1:
Create an instance of the com.lowagie.text.Document-object:
Document document = new Document();

Step 2:
Create a RtfWriter2 that listens to this document and writes the document to the OutputStream of your choice:
RtfWriter2.getInstance(document, new FileOutputStream("Chap0801.rtf"));

Step 3:
Open the document:
document.open();

Step 4:
Add content to the document:
document.add(new Paragraph("Hello World"));

Step 5:
Close the document:
document.close();

For help on how to create different objects and adding them to the document please look at the other sections of this tutorial.

Unsupported Features
These features are not supported by the RTF package. Everything else should work.
  • Watermarks
  • Viewer preferences
  • Encryption
  • Embedded fonts
  • Phrases with a leading
  • Lists with non-bullet symbols
  • Nested tables
  • Images other than JPEG, PNG, BMP and WMF
  • Rotated images
Extended Headers and Footers in RTF
With the RtfWriter2 it is not possible to change the header or footer for the next page just by calling setHeader before calling newPage(). There are two solutions to this problem which can be combined:
  • Using Chapters: You can define a different header or footer for each Chapter. Simply use the normal document.setHeader(HeaderFooter) or document.setFooter(HeaderFooter) methods to set the correct header or footer before you add the Chapter to the document.
  • Using the RtfHeaderFooterGroup: The RtfHeaderFooterGroup (in the com.lowagie.text.rtf.headerfooter package) allows you to define different headers or footers for different pages. Simply create a RtfHeaderFooterGroup and then use the setHeaderFooter(HeaderFooter, position) method to set the correct headers or footers for the different positions and then set the RtfHeaderFooterGroup as header or footer. Valid positions are DISPLAY_FIRST_PAGE, DISPLAY_LEFT_PAGES, DISPLAY_RIGHT_PAGES, DISPLAY_ALL_PAGES.

    This example creates a header that only displays on the title page:

    RtfHeaderFooterGroup headerGroup = new RtfHeaderFooterGroup();
    headerGroup.setHeaderFooter(new RtfHeaderFooter(new Phrase("This is a header on the title page")), com.lowagie.text.rtf.headerfooter.RtfHeaderFooter.DISPLAY_FIRST_PAGE);
    document.setHeader(headerGroup);
For a detailed example using Chapters look at Example 2.
For a detailed example using the RtfHeaderFooterGroup look at Example 3.
Table of Content in RTF
You can add a table of contents to the rtf document by adding an instance of the RtfTableOfContents class to your document. The RtfTableOfContents is a Chunk, so it's best to wrap it in a Paragraph:

RtfTableOfContents toc = new RtfTableOfContents("RIGHT-CLICK HERE AND THEN SELECT \"UPDATE FIELD\" TO UPDATE");
Paragraph para = new Paragraph();
para.add(toc);
document.add(para);

The RtfTableOfContents constructor takes a String as a parameter. This text is displayed when the user opens the rtf document. The user then has to update the table of contents himself, since the RtfWriter2 can not determine which entries exist during the document creation process.

After creating the table of contents add RtfTOCEntry objects wherever you want to have table of contents entries.

document.add(new RtfTOCEntry("This is a TOC entry", new Font()));

Alternatively you can use the setAutogenerateTOCEntries(autogenerate) method of the RtfWriter2 to instruct the RtfWriter2 to automatically add RtfTOCEntry objects for each Chapter and Section you add.

See Example 4 for a more detailed example.
