Part III: Advanced PDF

Chapter 9: Fonts
Fonts
This chapter deals with different types of fonts. If you only need simple Type 1 fonts (Courier, Helvetica, Times Roman, Symbol, ZapfDingbats), please take a look at Chapter 2.
Specifying an encoding
In the following example we will use a known font (Helvetica) specifying an encoding that is widely used in Europe: 'Cp1252' (this is latin 1 extended with 27 characters; the encoding is also known as 'winansi').
BaseFont helvetica = BaseFont.createFont("Helvetica", "Cp1252", false);
Font font = new Font(helvetica, 12, Font.NORMAL);
Chunk chunk = new Chunk("Sponsor this example and send me 1\u20ac.", font);
document.add(chunk);
 As you can see, the EURO symbol is added by using its UNICODE value. Check out the result.
True Type Fonts
In example 2, we are going to use a very popular True Type font:
BaseFont bfComic = BaseFont.createFont("c:\\windows\\fonts\\comic.ttf", "winansi", false);
Font FontComic = new Font(bfComic, 12, Font.NORMAL);
Chunk chunk = new Chunk("This is the quite popular True Type font 'Comic'.", FontComic);
document.add(chunk);
As you can see, you need to create a font using the path to the ttf-file. The above example works on a Windows 98 machine, but if you use another OS, you will have to change this path. Also, if the used TrueType font is not found on your machine, Acrobat Reader will throw an error such as: 'a font is missing or corrupted'. This can be the case when you try to open Chap0902.pdf.

That is why it's safer to embed the font in the PDF file, by setting the last parameter of method createFont to true:
BaseFont bfComic = BaseFont.createFont("c:\\windows\\fonts\\comic.ttf", "winansi", true);
Font FontComic = new Font(bfComic, 12, Font.NORMAL);
Chunk chunk = new Chunk("This is the quite popular True Type font 'Comic'.", FontComic);
document.add(chunk);
As you can see in example 3, the resulting PDF file is a lot larger: it contains all Acrobat Reader needs to know to show the TrueType font.
Asian Fonts
You can also add text in Asian fonts (using UNICODE):
BaseFont bfChinese = BaseFont.createFont("STSong-Light", "UniGB-UCS2-H", false);
Font FontChinese = new Font(bfChinese, 12, Font.NORMAL);
Chunk chunk = new Chunk("\u6e96\u53d7\u4fdd\u4eba", FontChinese);
document.add(chunk);
 This piece of code is used in example4 and you can take a look a the result here.

Important:
You can't embed these Asian fonts with iText. Acrobat can do it and so can ghostscript but iText can't. Embedding fonts is not trivial, embedding CJK fonts is even less trivial. In iText either you use the Adobe CJK fonts or use another library (I never saw a library that could do this but it probably exists). Before trying to view a pdf with Asian characters, you must download the Asian font pack from Adobe at: http://www.adobe.com/products/acrobat/acrrasianfontpack.html

You can create PDF-documents with the following fonts:
  • Chinese Simplified
    STSong-Light with the encodings UniGB-UCS2-H and UniGB-UCS2-V
  • Chinese traditional
    MHei-Medium and MSung-Light with the encodings UniCNS-UCS2-H and UniCNS-UCS2-V
  • Japanese
    HeiseiKakuGo-W5 and HeiseiMin-W3 with the encodings UniJIS-UCS2-H,UniJIS-UCS2-V,UniJIS-UCS2-HW-H and UniJIS-UCS2-HW-V
  • Korean
    HYGoThic-Medium and HYSMyeongJo-Medium with the encodings UniKS-UCS2-H and UniKS-UCS2-V


Barcodes
Example 5 is a very specific example in which we are going to use a special True Type font. iText has a class called PdfBarcode that can be used to generate 4 types of barcodes: CODE 39, UPC-A, EAN-13 and Interleaved 2/5. This class has to be used in combination with some specific TTF-files that can be downloaded from http://www.lowagie.com/iText/downloads/barcodefonts.zip (not all these fonts are free; please read the txt-files first before you use any of these fonts).
The PdfBarcode-object can be constructed like this: Chunk chunk = new PdfBarcode("c:\\winnt\\fonts\\CODE39.TTF", PdfBarcode.CODE39, 36, "0123456789")
As you can see, you need to tell the system where you have put the ttf-file and you also need to mention the type. If you want to use another barcode font than the ones provided, you will have to make some changes to the class PdfBarcode!
The third and fourth parameter are the fontsize and finally the number you want to turn into a barcode.
For UPC, EAN en I2/5, you can choose between different styles of fonts: see example 5 and the resulting PDF file.
