What is PDF?
PDF is the 'Portable Document Format', i.e. the native file format of the Adobe(R) Acrobat(R) family of products. The goal of these products is to enable users to exchange and view electronic documents easily and reliably, independent of the environment in which they were created.
Check out the Adobe website where you can download some White Papers and the Portable Document Format Reference Manual for more information.

I have troubles downloading the source/jarfile from the ftp-site.
Maybe you're behind a Firewall or IP-masquerading setup. In this case, you have to change your FTP-settings into 'passive mode'. You can do this by typing 'passive' before you perform a 'get' or by checking the option 'passive' if you use some GUI-program to FTP.

Problems with the Internet Explorer plugin
There is a bug in MSIE that still has to be fixed by the developers at MicroSoft. When you send dynamically created CORRECT (!) PDFs, you only get a blank browser window. You can use the following workaround to avoid this:

 res.setContentType("application/pdf");
 ByteArrayOutputStream ba = new ByteArrayOutputStream();
 generate_pdf(ba);
 res.setContentLength(ba.size());
 ServletOutputStream out = res.getOutputStream();
 ba.writeTo(out);
 out.flush();

If you don't include the content lenght it only works sometimes, if ever.

How to make a page in landscape
In most of the examples, the pages are created with a PageSize.A4. These pages are shown in portrait. If you create a page with PageSize.A4.rotate(), the pages are shown in landscape. You can also use the setPageSize-method, but be aware of the fact that the PageSize will only be changed on the next page.

On the subject of images!
iText supports images (plain GIFs, JPEGs and PNGs), but there are some limitations!
    These are the exceptions:
  • GIF
    • Gifs without a global colortable
    • interlaced gifs or gifs using local color table
    • ... (check out the error message!)
  • JPG
    • JPG with of which the number of 'Bits Per Component' != 8
    • JPG with 'invalid JPEG markers' (see the code of class Jpeg)
  • PNG
    • PNGs with BitDepth == 16
    • PNGs with another colortype than 0, 2 or 3
    • interlaced PNGs


How do I generate documentation files (the API)?
I have generated the API-pages using the javadoc tool that comes with the JDK. This is the command line I used on Windows: 
javadoc -use -windowtitle iText -doctitle "Java-Pdf-Html Library"
-link http://java.sun.com/products/jdk/1.2/docs/api -classpath .
-version -author -private -d D:/docs com.lowagie.text com.lowagie.text.pdf
com.lowagie.text.pdf.font com.lowagie.text.html com.lowagie.text.xml
When I compile the source code I get ClassNotFound errors!
If the missing classes are for instance java.util.Iterator or java.util.ArrayList, this is a sign that your are not using the JDK1.2. You should upgrade your JDK or use the 1.1.x-version of the library.

Can I use iText with the JDK1.1?
Yes, the original code was written for the JDK1.2, but it was ported to the JDK 1.1.x by Volker Richert. You can download the source from my FTP-site. (Remark: This 1.1 compatible version was tested by Rico Kahnert.)
There is also a 1.1.x port of iText0.35 by Paulo Soares.
Important: the zip- and jar-file only contain the SOURCE CODE. You will have to unzip/unjar and compile this code first! You will also need to download the Collection classes to make the library work (more info)
